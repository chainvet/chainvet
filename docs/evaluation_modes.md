# Mode Scoring (Precision / Recall)

This repo now includes a script to evaluate `--static`, `--symbolic`, `--fuzzing`, and `--hybrid` on the same contract against a ground-truth label set.

## Script

- `scripts/score_modes.sh`

## Ground Truth Format

JSON file:

```json
{
  "fixture": "fixtures/chatgpttestcases.sol",
  "truth": [
    "tx-origin",
    "delegatecall",
    "unchecked-call",
    "selfdestruct",
    "timestamp",
    "shadowing",
    "reentrancy"
  ]
}
```

Example file included:

- `fixtures/ground_truth/chatgpttestcases.json`

## Run

```bash
scripts/score_modes.sh fixtures/chatgpttestcases.sol fixtures/ground_truth/chatgpttestcases.json
```

The script runs all 4 modes, extracts finding kinds, maps them to canonical classes, and prints:

- `tp`, `fp`, `fn`
- `precision`, `recall`, `f1`
- label-level breakdown (`tp_labels`, `fp_labels`, `fn_labels`)

## Label Normalization Update (2026-03-09)

To reduce cross-mode name fragmentation before scoring:

- symbolic now emits `integer-underflow` instead of `underflow`
- fuzzing report output now emits canonical labels for selected aliases:
  - `tx-origin-auth` -> `tx-origin`
  - `hardcoded-gas` -> `hardcoded-gas-transfer`
  - `storage-memory-issue` -> `memory-manipulation`
- hybrid ingestion now stores canonical fuzz labels in `runs/<run_id>/findings.json`

## Rule-Set Expansion Update (2026-03-09)

- symbolic added:
  - `weak-prng`
  - `hardcoded-gas-transfer`
  - `unsafe-send-in-require`
  - `unprotected-ether-withdrawal`
  - `dos-with-failed-call`
  - `transaction-order-dependency`
  - `signature-malleability`
- fuzzing `check_all` now also includes:
  - `exception-disorder`

  - `locked-ether`
  - `unsafe-send-in-require`
  - plus AST-level `public-mint-burn`
  - plus `dos-with-failed-call`
  - plus `transaction-order-dependency`
  - plus `signature-malleability`

FP tuning applied:

- `transaction-order-dependency` and `signature-malleability` are now static-gated for symbolic/fuzzing parity outputs.

Confidence tags:

- Fuzzing text output now prints `[kind] [severity] [confidence]`.
- Symbolic detailed output now prints `confidence=<tier>` per finding.

These updates improve recall but may affect precision; re-run scoring after each expansion.

## Notes

- This is a label-level scorer (set-based), not path-level exploit validation.
- Unknown/non-target findings are counted as `other:<kind>` and penalize precision.
- `--hybrid` kinds are read from `runs/<run_id>/findings.json` generated by hybrid mode.
